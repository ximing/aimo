# 草稿自动保存功能

## 功能概述

为了防止用户在创建或编辑 memo 时因刷新页面、意外关闭标签页等原因导致内容丢失，我们实现了草稿自动保存功能。

## 功能特性

### 1. 自动保存

- **触发时机**：当用户输入内容后，系统会在 1 秒延迟后自动保存草稿到 localStorage
- **保存内容**：
  - 笔记内容（content）
  - 选中的类别（categoryId）
  - 公开/私密状态（isPublic）
  - 已选标签（tags）
  - 已上传的附件（attachments）
  - 关联的笔记（relations）
  - 保存时间戳（timestamp）

### 2. 草稿恢复

- **恢复时机**：页面刷新或重新打开编辑器时
- **用户确认**：检测到草稿后会弹窗询问用户是否恢复
  - 点击"确定"：恢复所有草稿内容
  - 点击"取消"：丢弃草稿，使用初始状态
- **恢复提示**：成功恢复草稿后会显示提示消息

### 3. 草稿清除

- **保存成功后**：成功保存 memo 后自动清除对应草稿
- **用户拒绝恢复**：用户选择不恢复草稿时清除
- **自动过期**：草稿保存超过 7 天后自动过期并清除

### 4. 草稿隔离

- **创建模式**：使用固定 key `memo_draft_create`
- **编辑模式**：使用 `memo_draft_edit_{memoId}` 格式的 key
- 不同 memo 的草稿互不干扰

## 技术实现

### DraftService

位置：`apps/web/src/services/draft.service.ts`

主要方法：

- `saveDraft(mode, data, memoId?)` - 保存草稿
- `getDraft(mode, memoId?)` - 获取草稿
- `clearDraft(mode, memoId?)` - 清除草稿
- `hasDraft(mode, memoId?)` - 检查是否存在草稿
- `clearExpiredDrafts()` - 清除所有过期草稿

### 集成位置

1. **MemoEditorForm** (`apps/web/src/components/memo-editor-form.tsx`)
   - 初始化时恢复草稿
   - 内容变化时自动保存
   - 保存成功后清除草稿

2. **App** (`apps/web/src/App.tsx`)
   - 应用启动时清除过期草稿

## 使用场景

### 场景 1：创建新笔记时意外刷新

1. 用户开始创建新笔记，输入了一些内容
2. 用户不小心刷新了页面
3. 页面重新加载后，系统检测到草稿并弹窗询问
4. 用户确认恢复，之前输入的内容全部恢复

### 场景 2：编辑笔记时浏览器崩溃

1. 用户正在编辑一个笔记，做了大量修改
2. 浏览器意外崩溃
3. 重新打开浏览器并进入编辑页面
4. 系统提示恢复草稿，用户确认后继续编辑

### 场景 3：切换标签页后忘记保存

1. 用户在编辑笔记时切换到其他标签页
2. 过了一段时间后回来，不确定是否保存了
3. 刷新页面后系统提示有草稿
4. 用户选择恢复，继续完成编辑

## 注意事项

1. **localStorage 限制**：草稿存储在浏览器的 localStorage 中，受浏览器存储配额限制
2. **隐私模式**：在浏览器隐私模式下，localStorage 可能在关闭窗口后被清除
3. **跨设备**：草稿不会在不同设备间同步
4. **附件处理**：草稿只保存已上传的附件信息，未上传的文件不会保存

## 未来改进方向

1. **云端同步**：将草稿同步到服务器，支持跨设备访问
2. **版本历史**：保存多个草稿版本，支持恢复到任意历史版本
3. **冲突解决**：当草稿和服务器数据冲突时，提供更好的合并策略
4. **自动保存指示器**：显示"已自动保存"状态提示
5. **草稿列表**：提供草稿管理界面，查看和管理所有草稿
