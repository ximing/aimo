## Codebase Patterns
- Use Drizzle ORM for MySQL schema - all schemas in schema/ directory must be exported from index.ts
- Always build server (`pnpm build`) before generating migrations (`pnpm migrate:generate`)
- Migration files are generated in drizzle/ folder at server root
- Controllers in apps/server/src/controllers/v1/ are auto-registered via glob import in controllers/index.ts
- Services use @Service() decorator for TypeDI injection
- Frontend services use @rabjs/react Service pattern with observable state
- Register new frontend services in apps/web/src/main.tsx via register()
- Register new backend controllers in apps/server/src/controllers/index.ts
- Use `sql` template for bigint comparisons in Drizzle queries
- Added `deletedAt` field to MemoListItemDto for trash functionality
