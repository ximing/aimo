## Codebase Patterns
- This project uses BOTH LanceDB (vector database) and MySQL (via Drizzle ORM) in a hybrid architecture
- LanceDB schemas are defined in `apps/server/src/models/db/schema.ts` using Apache Arrow types
- Drizzle/MySQL schemas are defined in `apps/server/src/db/schema/*.ts`
- When adding fields to tables, BOTH schemas must be updated
- LanceDB migrations are in `apps/server/src/migrations/scripts/` and run automatically on startup
- Drizzle migrations are generated via `pnpm migrate:generate` after building
- For bigint fields: LanceDB uses `Int64()`, Drizzle uses `bigint('field_name', { mode: 'number' })`
- Type interfaces for LanceDB records use `bigint` TypeScript type (not `number`)
- **Soft delete pattern**: All queries must filter `deletedAt = 0` using `and(eq(table.field, value), eq(table.deletedAt, 0))`
- Authentication middleware: Check `user.deletedAt > 0` and return "Account has been deleted" error
